================================================================================
  SARIA SELF-SUFFICIENCY REFACTOR PLAN
  Last updated: start of refactor arc
================================================================================

GOAL
----
Remove Saria's dependency on short-lived child projectiles for visual feedback
and game logic. All effects should be handled internally inside Saria.cs and
SariaIdleAnimator.cs. Child projectiles that exist purely as "timers" or
"floating bubbles" should be replaced with internal fields and draw calls.

This makes Saria cheaper (fewer projectile slots), more predictable in
multiplayer, and easier to maintain.

================================================================================
  REFACTOR PHASES
================================================================================

PHASE 1 — Bubble Face Removal  ← START HERE (next conversation)
  Target: Sad, Happiness, Notice
  Plan:
    1. Add timer fields to SariaIdleAnimator:
         _sadBubbleTimer, _happyBubbleTimer, _noticeBubbleTimer (int, count-down)
    2. Add public Set methods: SetSadBubble(int ticks), SetHappyBubble, SetNoticeBubble
    3. Add draw calls in SariaIdleAnimator for each bubble (load texture once,
       draw at head offset with alpha fade in last ~60 ticks)
    4. Include fields in Write/Read for net sync
    5. In Saria.cs, replace every NewProjectile call for Sad/Happiness/Notice
       with calls to IdleAnimator.SetSadBubble() etc.
    6. Verify build. Test in-game.

PHASE 2 — PowerUp / LocatorHitVisual / EmptyCup evaluation
  Examine each for gameplay vs visual-only role.
  Internalize the visual-only ones.

PHASE 3 — FormChangeOverlay
  Move overlay draw into Saria's own PreDraw/PostDraw if it has no hitbox.

PHASE 4 — HealCursor / Ztarget
  Low priority. Revisit after phases 1-3.

================================================================================
  KNOWN LATENT BUG
================================================================================

  _periodTimer is NOT included in SendExtraAI / ReceiveExtraAI.
  This means in multiplayer the period timer state is not synced to other clients.
  Fix: add `writer.Write(_periodTimer);` / `_periodTimer = reader.ReadInt32();`
  to the SendExtraAI / ReceiveExtraAI blocks in Saria.cs.
  Priority: fix this early in Phase 1 alongside the bubble timer additions.
